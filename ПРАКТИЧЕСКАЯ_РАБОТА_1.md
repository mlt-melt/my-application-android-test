```java
float density = displayMetrics.density;          // Плотность пикселей

// ПРЕОБРАЗОВАНИЕ В DP
int screenWidthDP = (int) (screenWidth / density);
int screenHeightDP = (int) (screenHeight / density);
```


#### 13.2: Сдвиг элементов (Bias)
```xml
<!-- Сдвиг вправо на 75% -->
<TextView
    app:layout_constraintStart_toStartOf="parent"
    app:layout_constraintEnd_toEndOf="parent"
    app:layout_constraintHorizontal_bias="0.75" />

<!-- Сдвиг вниз на 75% -->
<TextView
    app:layout_constraintTop_toBottomOf="@id/other"
    app:layout_constraintBottom_toBottomOf="parent"
    app:layout_constraintVertical_bias="0.75" />
```

#### 13.3: Горизонтальная цепочка элементов
```xml
<!-- Режим: spread (с промежутками) -->

    android:id="@+id/chain_h_1"
    app:layout_constraintStart_toStartOf="parent"
    app:layout_constraintEnd_toStartOf="@id/chain_h_2"
    app:layout_constraintHorizontal_chainStyle="spread" />

<!-- Режим: packed (плотно) -->
<TextView
    android:id="@+id/chain_packed_1"
    app:layout_constraintHorizontal_chainStyle="packed" />
```

#### 13.4: Вертикальная цепочка элементов
```xml

    android:id="@+id/chain_v_1"

    app:layout_constraintBottom_toTopOf="@id/chain_v_2"
```

#### 13.5: Программное создание ConstraintLayout
```java
// Создание контейнера
ConstraintLayout constraintLayout = new ConstraintLayout(this);
constraintLayout.setLayoutParams(new ViewGroup.LayoutParams(
    ViewGroup.LayoutParams.MATCH_PARENT,
    ViewGroup.LayoutParams.MATCH_PARENT
));

// Создание элемента
TextView titleView = new TextView(this);
titleView.setId(1001); // Важно установить ID
titleView.setText("Заголовок");

// Добавление в контейнер
constraintLayout.addView(titleView);

// Применение ограничений через ConstraintSet
ConstraintSet constraintSet = new ConstraintSet();
constraintSet.connect(titleView.getId(), ConstraintSet.TOP, 
                      ConstraintSet.PARENT_ID, ConstraintSet.TOP, 16);
constraintSet.connect(titleView.getId(), ConstraintSet.START,
                      ConstraintSet.PARENT_ID, ConstraintSet.START, 0);
constraintSet.connect(titleView.getId(), ConstraintSet.END,
                      ConstraintSet.PARENT_ID, ConstraintSet.END, 0);

// Создание горизонтальной цепочки в коде
constraintSet.createHorizontalChain(
    ConstraintSet.PARENT_ID, ConstraintSet.START,
    ConstraintSet.PARENT_ID, ConstraintSet.END,
    new int[]{item1.getId(), item2.getId(), item3.getId()},
    null,
    ConstraintSet.CHAIN_SPREAD);

constraintSet.applyTo(constraintLayout);
```

## 🎓 Ключевые концепции

### Размеры в Android:
- **px** (пиксели) - 1:1 соответствие физическому пикселю
- **dp** (device-independent pixels) - масштабируется в зависимости от плотности экрана
- **match_parent** - занимает всю доступную ширину/высоту родителя
- **wrap_content** - размер по содержимому

### Отступы:
- **Padding** - внутренний отступ (между содержимым и границей элемента)
- **Margin** - внешний отступ (между элементом и его соседями)

### ConstraintLayout преимущества:
- Гибкое позиционирование без вложенности
- Цепочки элементов для создания рядов/столбцов
- Bias для сдвига элементов
- Лучшая производительность чем вложенные LinearLayout

---

## 📝 Примечания

- Все размеры в коде рекомендуется хранить в dp и преобразовывать в px через `density`
- При создании View в коде обязательно устанавливайте ID (setId) для ConstraintLayout
- В ConstraintLayout используйте `app:` namespace для constraint атрибутов
- Lifecycle методы Activity помогают управлять ресурсами (например, остановка сервисов при onPause)
